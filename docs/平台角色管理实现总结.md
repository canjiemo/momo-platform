## 🏗️ 架构设计

### 数据流向

```
平台管理员
    ↓
创建/更新平台角色 (public.sys_role)
    ↓
配置菜单权限 (public.sys_role_menu)
    ↓
    ├─→ [自动触发] 创建新租户时自动同步 (基于 feature_level 筛选)
    │       ↓
    │   复制到 {tenant}.sys_role (含 platform_role_id)
    │       ↓
    │   同步角色-菜单关联
    │       ↓
    │   记录同步关系 (public.sys_tenant_role)
    │
    └─→ [手动触发] 同步到已有租户
            ↓
        复制到 {tenant}.sys_role (含 platform_role_id)
            ↓
        同步角色-菜单关联
            ↓
        记录同步关系 (public.sys_tenant_role)

平台角色更新
    ↓
[自动触发] 同步到所有已分配租户
    ↓
更新 {tenant}.sys_role
    ↓
重新同步角色-菜单关联

租户用户尝试修改/删除平台同步的角色
    ↓
Service 层检查 platform_role_id
    ↓
抛出 BusinessException (只读保护)
```

### 关键表结构

#### public.sys_role (平台角色表)
```sql
CREATE TABLE public.sys_role (
    id BIGSERIAL PRIMARY KEY,
    role_name VARCHAR(50) NOT NULL,
    description VARCHAR(255),
    role_type SMALLINT,              -- 1=平台专用 2=租户模板
    feature_level SMALLINT,          -- 1=基础版 2=标准版 3=企业版
    status SMALLINT DEFAULT 1,
    delete_flag SMALLINT DEFAULT 0,
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);
```

#### {tenant}.sys_role (租户角色表)
```sql
CREATE TABLE sys_role (
    id BIGSERIAL PRIMARY KEY,
    role_name VARCHAR(50) NOT NULL,
    description VARCHAR(255),
    role_type SMALLINT,
    feature_level SMALLINT,
    status SMALLINT DEFAULT 1,
    delete_flag SMALLINT DEFAULT 0,
    platform_role_id BIGINT,         -- 平台角色ID(非NULL表示平台同步,只读)
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);
```

#### public.sys_tenant_role (同步关系表)
```sql
CREATE TABLE public.sys_tenant_role (
    id BIGSERIAL PRIMARY KEY,
    tenant_id BIGINT NOT NULL,           -- 租户ID
    platform_role_id BIGINT NOT NULL,    -- 平台角色ID
    tenant_role_id BIGINT NOT NULL,      -- 租户schema中的角色ID
    synced_at TIMESTAMP NOT NULL,        -- 同步时间
    synced_by BIGINT,                    -- 同步人
    UNIQUE(tenant_id, platform_role_id)  -- 防止重复同步
);
```

### 权限分配建议

- **平台超级管理员**: 拥有所有 `platform:*` 权限
- **平台角色管理员**: `platform:role:*` 权限
- **平台运维人员**: `platform:tenant:role:sync` 权限
- **租户用户**: 只能查看和使用角色,不能修改平台同步的角色

## 📊 Feature Level 分级

| Level | 名称 | 说明 | 同步规则 |
|-------|------|------|---------|
| 1 | 基础版 | 入门级功能,适合小型组织 | 同步 `feature_level <= 1` 的模板 |
| 2 | 标准版 | 标准功能,适合中型组织 | 同步 `feature_level <= 2` 的模板 |
| 3 | 企业版 | 全功能,适合大型组织 | 同步所有模板 (`feature_level <= 3`) |

**示例**:
```sql
-- 平台配置角色模板
INSERT INTO public.sys_role (role_name, role_type, feature_level) VALUES
    ('普通员工', 2, 1),      -- 基础版、标准版、企业版都有
    ('部门主管', 2, 2),      -- 标准版、企业版才有
    ('系统管理员', 2, 3);    -- 仅企业版

-- 创建基础版租户(feature_level=1)
-- 自动同步: 普通员工
-- 不同步: 部门主管、系统管理员

-- 创建企业版租户(feature_level=3)
-- 自动同步: 普通员工、部门主管、系统管理员
```

## 🔄 同步机制

### 1. 自动同步 (创建租户时)

**触发时机**: `TenantSchemaService.initializeTenantSchema()` 步骤5

**同步内容**:
- 菜单模板 (`menu_type=2` 且 `feature_level <= 租户级别`)
- 角色模板 (`role_type=2` 且 `feature_level <= 租户级别`)
- 角色-菜单关联关系

**特点**:
- 无需手动干预
- 根据租户版本自动筛选
- 空模板不报错(优雅降级)

### 2. 手动同步

**触发方式**: 调用同步接口

**同步方式**:
- 单个同步: `/platform/tenant/role/sync`
- 批量同步: `/platform/tenant/role/sync/batch`
- 全量同步: `/platform/tenant/role/sync/all`

**特点**:
- 灵活控制同步时机
- 支持批量操作
- 幂等性保护(已同步则跳过)

### 3. 更新传播 (平台角色更新时)

**触发时机**: `PlatformRoleService.update()` 或 `assignMenus()`

**传播范围**: 所有已分配该角色的租户

**更新内容**:
- 角色基本信息(名称、描述、状态)
- 菜单权限配置

**特点**:
- 全自动,无需手动触发
- 仅对 `role_type=2` 的租户模板角色生效
- 保证租户数据与平台同步
