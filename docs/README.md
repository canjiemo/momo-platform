# Seer Fitness Edu - 系统架构与模块说明

## 📐 系统架构

### 整体架构
Seer Fitness Edu 是一个基于 **RBAC (Role-Based Access Control)** 的健身教育管理系统,采用前后端分离架构。

```
┌─────────────────────────────────────────────────────────────┐
│                        前端应用层                              │
│                  (Vue/React + Ant Design)                    │
└───────────────────────┬─────────────────────────────────────┘
                        │ RESTful API
┌───────────────────────▼─────────────────────────────────────┐
│                      后端服务层                               │
│                   (Spring Boot)                              │
├─────────────────────────────────────────────────────────────┤
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│  │  认证授权     │  │  业务逻辑     │  │  日志审计     │      │
│  │  模块        │  │  模块        │  │  模块        │      │
│  └──────────────┘  └──────────────┘  └──────────────┘      │
└───────────────────────┬─────────────────────────────────────┘
                        │ JDBC
┌───────────────────────▼─────────────────────────────────────┐
│                      数据持久层                               │
│                    (PostgreSQL)                              │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐       │
│  │ 用户表    │ │ 角色表    │ │ 菜单表    │ │ 日志表    │       │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘       │
└─────────────────────────────────────────────────────────────┘
```

### 技术栈
- **后端**: Spring Boot + MyBatis/JPA + PostgreSQL
- **前端**: Vue.js/React + Ant Design
- **认证**: JWT/Session
- **权限**: RBAC (基于角色的访问控制)
- **密码加密**: BCrypt

## 🏗️ 核心模块说明

### 1. 系统管理模块 (System Management)

#### 1.1 用户管理 (User Management)
**模块路径**: `/system/user`
**核心功能**:
- 管理系统中的所有用户账号(运维人员、教师、学生)
- 支持用户类型区分: `user_type` (0-运维 1-教师 2-学生)
- 用户的增删改查操作
- 密码初始化和重置功能
- 用户状态管理(启用/禁用)

**权限点**:
- `user:view` - 查看用户列表和详情
- `user:create` - 创建新用户
- `user:update` - 编辑用户信息
- `user:delete` - 删除用户(逻辑删除)
- `user:init-password` - 初始化用户密码
- `user:reset-password` - 重置用户密码

**业务场景**:
- 运维人员创建教师账号
- 教师创建学生账号
- 忘记密码时的重置操作

---

#### 1.2 角色管理 (Role Management)
**模块路径**: `/system/role`
**核心功能**:
- 定义系统中的角色(如:系统管理员、教师、学生等)
- 角色的增删改查
- 为角色分配菜单权限

**权限点**:
- `role:view` - 查看角色列表
- `role:create` - 创建新角色
- `role:update` - 编辑角色信息
- `role:delete` - 删除角色
- `role:assign` - 为角色分配菜单权限

**业务场景**:
- 创建"教学主管"角色,分配课程管理权限
- 创建"财务人员"角色,分配收费管理权限

---

#### 1.3 菜单管理 (Menu Management)
**模块路径**: `/system/menu`
**核心功能**:
- 管理系统的菜单树结构
- 支持三种类型: 目录(0)、菜单(1)、按钮(2)
- 配置前端路由和权限字符
- 菜单排序和图标配置

**权限点**:
- `menu:view` - 查看菜单树
- `menu:create` - 创建菜单/按钮
- `menu:update` - 编辑菜单配置
- `menu:delete` - 删除菜单

**业务场景**:
- 新增业务模块时,添加对应菜单
- 调整菜单显示顺序
- 配置按钮级权限控制

---

#### 1.4 组织管理 (Organization Management)
**模块路径**: `/system/organization`
**核心功能**:
- 管理公司/学校的组织架构(部门、分校等)
- 支持树形层级结构
- 配置组织负责人、联系方式、地址等
- 组织启用/禁用管理

**权限点**:
- `organization:view` - 查看组织架构树
- `organization:create` - 创建部门/分校
- `organization:update` - 编辑组织信息
- `organization:delete` - 删除组织节点

**业务场景**:
- 健身连锁机构管理多个分店
- 按部门划分管理权限
- 数据权限隔离(用户只能看自己部门的数据)

---

### 2. 日志管理模块 (Log Management)

#### 2.1 操作日志 (Operation Log)
**模块路径**: `/log/operation`
**核心功能**:
- 记录所有用户的操作行为
- 支持日志类型: CREATE/UPDATE/DELETE/QUERY/LOGIN/LOGOUT
- 记录请求参数、响应结果、IP地址、执行耗时
- 操作成功/失败状态跟踪
- 日志导出功能(Excel/CSV)

**权限点**:
- `operation_log:view` - 查看操作日志
- `operation_log:delete` - 删除历史日志
- `operation_log:export` - 导出日志数据

**业务场景**:
- 审计员查看系统操作记录
- 排查问题时追溯操作历史
- 定期导出日志进行归档
- 安全事件分析

**记录内容**:
```json
{
  "user": "张老师",
  "operation": "UPDATE",
  "module": "user",
  "desc": "修改学生李明的手机号",
  "ip": "192.168.1.100",
  "time": "2025-01-09 14:30:25",
  "duration": "125ms",
  "result": "成功"
}
```

### 3. 数据字典模块 (Data Dictionary)

#### 3.1 字典类型管理 (Dictionary Type)
**模块路径**: `/dict/type`
**核心功能**:
- 管理字典分类(如:性别、学历、课程类型等)
- 字典类型的增删改查
- 字典类型编码唯一性控制

**权限点**:
- `dict:type:view` - 查看字典类型
- `dict:type:create` - 创建字典类型
- `dict:type:update` - 编辑字典类型
- `dict:type:delete` - 删除字典类型

**示例数据**:
| 类型编码 | 类型名称 |
|---------|---------|
| gender  | 性别    |
| education | 学历  |
| course_type | 课程类型 |

---

#### 3.2 字典数据管理 (Dictionary Data)
**模块路径**: `/dict/data`
**核心功能**:
- 管理字典类型下的具体数据项
- 字典值的增删改查
- 字典排序和状态管理

**权限点**:
- `dict:data:view` - 查看字典数据
- `dict:data:create` - 创建字典数据
- `dict:data:update` - 编辑字典数据
- `dict:data:delete` - 删除字典数据

**示例数据**:
| 字典类型 | 字典标签 | 字典值 | 排序 |
|---------|---------|-------|-----|
| gender  | 男      | 1     | 1   |
| gender  | 女      | 2     | 2   |
| course_type | 瑜伽课 | yoga | 1 |
| course_type | 器械训练 | equipment | 2 |

**业务价值**:
- 前端下拉框数据源统一管理
- 避免硬编码,提高可维护性
- 支持动态配置,无需发版

## 🔐 RBAC 权限模型

### 权限控制流程

```
用户登录 → 查询用户角色 → 查询角色菜单权限 → 生成权限列表 → 前后端权限验证
```

### 核心表关系

```sql
sys_user (用户表)
    ↓ (N:M)
sys_user_role (用户角色关联表)
    ↓
sys_role (角色表)
    ↓ (N:M)
sys_role_menu (角色菜单关联表)
    ↓
sys_menu (菜单权限表)
```

### 特殊权限

#### 超级管理员 (admin_flag=1)
- 绕过所有权限检查
- 拥有系统所有功能访问权限
- 用于系统初始化和紧急维护

#### 数据权限(未来扩展)
- 本人数据权限
- 本部门数据权限
- 本部门及子部门数据权限
- 全部数据权限

## 🎯 典型应用场景

### 场景1: 健身房连锁管理
- **组织架构**: 总部 → 各分店 → 各部门
- **角色设置**: 总经理、店长、教练、前台、学员
- **权限分配**:
  - 总经理: 所有权限
  - 店长: 本店数据管理
  - 教练: 课程和学员管理
  - 前台: 学员查询和签到
  - 学员: 查看自己的课程

### 场景2: 在线健身教育平台
- **用户类型**: 运维人员、教师、学生
- **角色设置**: 平台管理员、课程管理员、教师、学生
- **权限分配**:
  - 平台管理员: 系统配置和用户管理
  - 课程管理员: 课程内容管理
  - 教师: 课程发布和学员管理
  - 学生: 学习和作业提交

## 🔧 使用说明

### 1. 初始化系统
```bash
# 1. 创建表结构
psql -U postgres -d seer_fitness -f docs/sql/rbac.sql

# 2. 导入测试数据
psql -U postgres -d seer_fitness -f docs/sql/data.sql
```

### 2. 登录测试
| 场景 | 用户名 | 密码 | 说明 |
|-----|-------|------|------|
| 超级管理员 | superadmin | Aa123456! | 所有权限 |
| 系统管理员 | sysadmin | Aa123456! | 通过角色获得所有权限 |
| 只读用户 | readonly | Aa123456! | 只能查看,不能操作 |
| 无权限用户 | noperm | Aa123456! | 测试拒绝访问 |

### 3. 开发接入
```java
// 后端权限注解示例
@RequiresPermissions("user:create")
public Result createUser(UserDTO dto) {
    // 创建用户逻辑
}

// 前端权限指令示例
<a-button v-if="$auth('user:delete')" @click="deleteUser">删除</a-button>
```
